# GRC-Portfolio
A structured GRC and ISO 27001 portfolio showcasing policies, standards, procedures, risk registers, compliance matrices, incident templates, audit checklists, and BIA documents. Built for learning and demonstrating practical cybersecurity governance skills.

## 📌 Included Artefacts

### Policies  
High-level rules that define organisational expectations.  
Examples included:
- Information Security Policy  
- Access Control Policy  
- Acceptable Use Policy  
- Vendor Management Policy  
- Data Privacy Policy  
- Change Management Policy  
- Backup & Recovery Policy  
- Third‑Party Risk Policy  

###  Standards  
Mandatory technical requirements supporting policies.  
Included:
- Password Standard  
- Encryption Standard  
- Endpoint Security Standard  

### Procedures  
Step-by-step instructions for operational tasks.  
Included:
- User Access Provisioning  
- Incident Response  
- Change Management  
- Backup Procedure  

### 📊 Risk Register  
A structured list of risks, likelihood, impact, controls, and ownership.

### 📚 Compliance Matrix  
ISO 27001 Annex A mapped to implemented controls.

### 🚨 Incident Report Template  
A structured template for documenting security incidents.

### 🔍 Audit Checklist  
Checklist for internal audits aligned with ISO 27001.

### 🏢 Business Impact Analysis (BIA)  
Identifies critical processes, impacts, and recovery priorities.
